Синхронизация данных между программами Демо: 1С:Библиотека стандартных подсистем, редакция 2.3 и Демо: 1С:Библиотека стандартных подсистем, версия 2.2.5 предназначена для демонстрации обратной совместимости, когда одна из программ разработана на базе редакции 2.3, а другая - на предыдущей версии 2.2.5. Синхронизация данных является демонстрационной и не предназначена для работы с реальными прикладными данными.

При отправке данных в программу версии 2.2.5

  • У справочника Виды контактной информации не передается вид контактной информации с типом Skype, который появился в версии 2.3.1.
    См. обработчик события ПередВыгрузкой ПКО Справочник: Виды контактной информации в правилах обмена (макет ПравилаОбмена).
  • У справочников Организации, Демо: Контрагенты, Демо: Партнеры и Пользователи также не передаются строки табличных частей Контактная информация с типом Skype и не передается колонка ВидДляСписка,  которая появилась в версии 2.3.1.
    См. алгоритм ВыгрузитьКонтактнуюИнформацию, используемый в обработчиках ПередВыгрузкой ПКО справочников-владельцев контактной информации, в правилах обмена (макет ПравилаОбмена).

  • в строках табличных частей Контактная информация записи с типом Skype сохраняются (не затираются).
    См. алгоритмы ВыгрузитьКонтактнуюИнформацию, ЗапомнитьОтсутствующуюКонтактнуюИнформацию и ЗагрузитьОтсутствующуюКонтактнуюИнформацию, вызываемые из обработчиков ПередВыгрузкой, ПриЗагрузке и ПослеЗагрузки соответственно,  в правилах обмена (макет ПравилаОбменаКорреспондента).

В какой из двух программ начинать процесс настройки

При настройке синхронизации данных между двумя программами необходимо определить в какой из двух программ начинать процесс настройки. Решение нужно принимать в зависимости от наполненности двух программ данными:

1. Если в обеих программах на момент начала настройки уже имеются данные, то в этом случае в качестве программы для начала настройки синхронизации данных лучше выбрать ту программу, в которой данные заполнены качественнее, более подробно.

При выборе необходимо учитывать, что если часть из имеющихся данных двух программ является общими (например, справочники Организации, Контрагенты и т.п.), то от того, какая программа выбрана в качестве источника, будет зависеть и то, к какому виду будет приведена эта часть общих данных.

Если не все общие данные нужно принять в том виде, в каком они содержатся в программе-источнике, то рекомендуется воспользоваться синхронизацией данных с установкой дополнительных параметров. При первой синхронизации данных необходимо загрузить из другой программы только те данные, которые должны быть загружены в том виде, в котором они хранились в ней. Остальные данные будут синхронизированы позже.

2. Если одна из программ (информационных баз) пустая или практически пустая, то выбор программы для начала процесса настройки упрощается. Стоит лишь учитывать то, что создание настройки синхронизации данных происходит быстрее в менее заполненной программе, т.к. при настройке тратится время на выгрузку данных.
