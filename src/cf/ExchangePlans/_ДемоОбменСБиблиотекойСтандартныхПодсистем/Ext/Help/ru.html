Этот план обмена предназначен для демонстрации обмена данными между однотипными программами библиотеки стандартных подсистем по правилам обмена. Использование правил обмена подразумевает гибкую настройку обмена и возможность реализации сложных сценариев обмена.

План обмена определяет состав объектов метаданных, которые участвуют в обмене данными. Если в составе плана обмена у объекта признак авторегистрации установлен в значение "Запретить", то это означает, что не все объекты данного типа регистрируются для выгрузки. Такие объекты попадают под действие правил регистрации.

Использование правил регистрации позволяет настраивать фильтрацию данных при обмене. Настройка ограничения миграции данных выполняется на закладке "Настройка фильтров регистрации":

  • параметр "Дата начала выгрузки документов" используется для ограничения выборки документов заданной датой; будут выгружены только документы, дата которых больше или равна установленной даты, например, с начала года;
  • параметры "Отбор по организациям", "Отбор по складам" и "Отбор по подразделениям" используются для включения ограничения миграции данных по заданному списку организаций, складов и подразделений соответственно. Если флаги сброшены, то ограничения к спискам не применяются;
  • в таблицах "Организации", "Склады" и "Подразделения" задаются списки элементов, которые ограничивают миграцию данных.


На закладке "Значения по умолчанию" указываются значения,которыми заполняются поля объектов при загрузке данных, если поля объектов не заполнены.

В этом плане обмена правила регистрации объектов применяются для следующих типов данных:
Справочник "Организации"
Справочник "Демо: Места хранения"
Справочник "Демо: Подразделения"
Документ "Демо: Поступление товаров"
Документ "Демо: Реализация товаров"
Регистр сведений "Демо: Ответственные лица"
Регистр сведений "Демо: Работники организаций"

Для элемента справочника регистрация изменений выполняется, если элемент принадлежит множеству элементов из соответствующей таблицы в узле.

Для документа регистрация изменений выполняется, если:

  • дата документа больше или равна параметру "Дата начала выгрузки документов" в узле;
  • и организация и склад и подразделение* принадлежат множеству элементов из одноименных таблиц в узле (при условии установки флагов использования ограничений по таблицам).

* Примечание: отбор по подразделениям используется только для документа "Демо: Реализация товаров".

Также план обмена демонстрирует работу дат запрета загрузки (подсистема Даты запрета изменения) для следующих типов данных:
Документ "Демо: Поступление товаров"
Документ "Демо: Реализация товаров"
Регистр сведений "Курсы валют"
