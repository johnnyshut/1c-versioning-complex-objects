Синхронизация данных с Демо: 1С:Библиотека стандартных подсистем, редакция 2.3 предназначена для демонстрации настройки и выполнения синхронизации данных между однотипными конфигурациями Демонстрационная конфигурация "Библиотека стандартных подсистем", редакция 2.3 по правилам обмена. Использование правил обмена позволяет производить гибкую настройку и реализовывать  сложные сценарии синхронизации данных. Создавать и редактировать правила обмена можно в конфигурации 1С: Конвертация данных, редакция 2.0. В этой синхронизации данных наглядно демонстрируются основные возможности подсистемы Обмен данными, такие как:

  • Использование различных вариантов подключения к другой программе.
  • Сопоставление объектов информационных баз, позволяющее указать, какие объекты являются одинаковыми для двух синхронизирующихся программ.
  • Синхронизация данных с использованием правил конвертации.
  • Настройка правил синхронизации данных, ограничивающих миграцию данных между двумя программами.
  • Автоматическая синхронизация данных по заранее заданному расписанию.

Синхронизация данных с Демонстрационная конфигурация "Библиотека стандартных подсистем", редакция 2.3 является демонстрационной и не предназначена для синхронизации реальных прикладных данных. Выполняется синхронизация следующих данных:

  • Нормативно-справочная информация: справочники Организации, Валюты, Виды контактной информации, Демо: Банковские счета, Демо: Места хранения, Демо: Подразделения и другие;
  • Два документа Демо: Поступление товаров и Демо: Реализация товаров;
  • а также регистры сведений Курсы валют, Демо: Ответственные лица и Демо: Работники организаций.

При настройке синхронизации необходимо настроить правила синхронизации данных, указав по каким организациям, подразделениям и складам (местам хранения) будут выгружаться данные. Также при настройке требуется задать значение номенклатуры по умолчанию, которое будет заполняться при загрузке документов, если в них значение номенклатуры не заполнено.

В качестве примера рассмотрим, как подсистема Обмен данными может применяться для реализации синхронизации данных между 1С:Управление торговлей, редакция 11.0 и 1С:Бухгалтерия предприятия, редакция 3.0. С ее помощью возможно следующее:

  • Повышение эффективности управления торговым предприятием;
  • Консолидацию данных управленческого и бухгалтерского учета;
  • Наличие в программе 1С:Управление торговлей, редакция 11.0 всех данных, необходимых для управления торговым предприятием;
  • Наличие в программе 1С:Бухгалтерия предприятия, редакция 3.0 всех данных, необходимых для регламентированного учета и сдачи отчетности;
  • Возможность работы сотрудников в привычной для себя среде:
    • бухгалтеры работают в программе 1С:Бухгалтерия предприятия, редакция 3.0 .
    • менеджеры работают в программе 1С:Управление торговлей, редакция 11.0.

В какой из двух программ начинать процесс настройки

При настройке синхронизации данных между двумя программами необходимо определить в какой из двух программ начинать процесс настройки. Решение нужно принимать в зависимости от наполненности двух программ данными:

1. Если в обеих программах на момент начала настройки уже имеются данные, то в этом случае в качестве программы для начала настройки синхронизации данных лучше выбрать ту программу, в которой данные заполнены качественнее, более подробно.

При выборе необходимо учитывать, что если часть из имеющихся данных двух программ является общими (например, справочники Организации, Контрагенты и т.п.), то от того, какая программа выбрана в качестве источника, будет зависеть и то, к какому виду будет приведена эта часть общих данных.

Если не все общие данные нужно принять в том виде, в каком они содержатся в программе-источнике, то рекомендуется воспользоваться синхронизацией данных с установкой дополнительных параметров. При первой синхронизации данных необходимо загрузить из другой программы только те данные, которые должны быть загружены в том виде, в котором они хранились в ней. Остальные данные будут синхронизированы позже.

2. Если одна из программ (информационных баз) пустая или практически пустая, то выбор программы для начала процесса настройки упрощается. Стоит лишь учитывать то, что создание настройки синхронизации данных происходит быстрее в менее заполненной программе, т.к. при настройке тратится время на выгрузку данных.
